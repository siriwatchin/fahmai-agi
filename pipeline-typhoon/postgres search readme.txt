# คู่มือการใช้งานระบบสืบค้นฐานข้อมูลเครื่องมือเอเจนต์ (postgres_search_tool)

ไลบรารี `postgres_search_tool.py` เป็นชุดเครื่องมือควบคุมและเชื่อมต่อฐานข้อมูลเชิงสัมพันธ์ PostgreSQL สำหรับระบบเอเจนต์ปัญญาประดิษฐ์ (AI Data-Agent) โดยได้รับการจัดหมวดหมู่อยู่ในรูปแบบแยกฟังก์ชันการทำงาน (Granular Functions) เพื่อลดความซับซ้อนของโครงสร้างพารามิเตอร์ ช่วยให้โมเดลภาษาขนาดเล็กถึงปานกลาง เช่น Pathumma-llm, Qwen และ Llama สามารถรับรู้ วิเคราะห์ และเลือกใช้งานเครื่องมือได้อย่างแม่นยำสูงสุด

---

## 1. คุณสมบัติหลักและระบบความปลอดภัย (Core Architecture & Security Features)

* **สถาปัตยกรรมแยกฟังก์ชัน (Granular Call Mapping):** แยกฟังก์ชันประมวลผลเชิงสถิติออกจากกันอย่างจำเพาะเจาะจงประกอบด้วย `count_rows`, `sum_column`, `find_max` และ `fetch_sample` เพื่อจำกัดขอบเขตการทำงานและลดภาระทางความคิดของโมเดลภาษา
* **การจำกัดสิทธิ์การเข้าถึงต่ำสุด (Least Privilege Connection):** บังคับใช้งานผ่านบัญชีการเชื่อมต่อที่มีสิทธิ์จำกัด ป้องกันการรันคำสั่งแก้ไขหรือทำลายข้อมูลในระบบ Production
* **การควบคุมสิทธิ์ระดับตาราง (Data Layer Whitelist Guard):** อนุญาตให้เข้าถึงเฉพาะรายชื่อตารางธุรกรรมหลัก 20 ตารางที่กำหนดไว้ในตัวแปร `self.ALLOWED_TABLES` เท่านั้น หากตรวจพบการเข้าถึงตารางนอกเหนือจากนี้ระบบจะปฏิเสธการทำงานทันที
* **การป้องกัน SQL Injection:** สกัดกั้นการโจมตีทางสคริปต์ด้วยกลไกทำความสะอาดชื่อตารางและคอลัมน์ (`_sanitize_identifier`) ร่วมกับการส่งผ่านค่าเงื่อนไขแบบ Parameterized Queries (`%s`)
* **การจัดการสลับโครงสร้างข้อมูลอัตโนมัติ (Automated Schema Cutover):** รองรับตรรกะการสลับชื่อคอลัมน์ส่วนลดตาราง `FACT_SALES` ระหว่างสคีมายุคเก่า (`discount_amt`) และยุคใหม่ (`discount_total_thb`) โดยประเมินจากพารามิเตอร์วันที่ (`filter_date`) อ้างอิงเกณฑ์จุดเปลี่ยนผ่านระบบวันที่ 1 เมษายน 2568 (`2025-04-01`)

## 2. การกำหนดค่าระบบสภาพแวดล้อม (`.env`)

จัดเตรียมไฟล์ชื่อ `.env` ไว้ในตำแหน่งโฟลเดอร์ปฏิบัติการหลักเดียวกันกับสคริปต์ทำงาน เพื่อกำหนดค่าการเชื่อมต่อฐานข้อมูลและสิทธิ์ API:

```text
TYPHOON_API_URL=[https://api.opentyphoon.ai/v1/chat/completions](https://api.opentyphoon.ai/v1/chat/completions)
TYPHOON_MODEL=typhoon-v2.5-30b-a3b-instruct # เปลี่ยนโมเดลที่ใช้
ANSWER_MODE=agent

# การตั้งค่าการเชื่อมต่อฐานข้อมูล PostgreSQL Core (Production Server)
DB_NAME=HACK4
DB_USER=admin
DB_PASSWORD=scamper
DB_HOST=swarm-manager.modelharbor.com
DB_PORT=50282
